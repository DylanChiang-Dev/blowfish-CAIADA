# 多语言内容开发工作流程

以繁体中文为模板，开发完成后同步到其他语言版本。

## 工作流程

### 1. 开发繁体中文版本
- 在 `*.zh-tw.md` 文件中完成所有内容和样式开发
- 确保布局、图片、样式都正确显示
- 请用户在 http://localhost:1313/zh-tw/ 预览确认

### 2. 用户确认
- 等待用户确认繁体中文版本无误
- 如有修改需求，继续在繁体中文版本上迭代

### 3. 同步到其他语言
用户确认后，将内容同步到以下语言版本：
- 简体中文 (`*.zh-cn.md`) - 转换繁体字为简体字
- 英文 (`*.en.md`) - 翻译为英文
- 越南文 (`*.vi.md`) - 翻译为越南文

### 4. 验证所有语言版本
- 确保所有语言的页面结构一致
- 确保图片路径正确引用
- 确保样式显示正常

## 文件命名规则

| 语言 | 后缀 | 示例 |
|------|------|------|
| 繁体中文 | `.zh-tw.md` | `_index.zh-tw.md` |
| 简体中文 | `.zh-cn.md` | `_index.zh-cn.md` |
| 英文 | `.en.md` | `_index.en.md` |
| 越南文 | `.vi.md` | `_index.vi.md` |

## 注意事项

- **繁体中文为主版本**：所有内容变更先在繁体中文版本完成
- **图片共享**：所有语言版本共享同一目录下的图片资源
- **保持结构一致**：各语言版本的 HTML 结构和 CSS 类名应保持完全一致
